# Usertest-Checkliste fuer echten Testlauf

Diese Checkliste ist fuer einen moderierten Testlauf mit echten Nutzerinnen und Nutzern gedacht.
Ziel ist, die realen Wege durch die App abzudecken und Auffaelligkeiten direkt im Dokument festzuhalten.

## So verwenden

- Hake Schritte ab, sobald sie im Test wirklich ausgefuehrt wurden.
- Trage Beobachtungen direkt unter dem jeweiligen Abschnitt ein.
- Halte besonders fest: Verwirrung, Abbrueche, Missverstaendnisse, Wartezeiten, Formulierungen, fehlendes Vertrauen, technische Fehler.

## Story 1: Brautpaar versteht die Landingpage

**User Story**
Als Brautpaar moechte ich auf der Startseite schnell verstehen, was das Produkt macht, damit ich entscheiden kann, ob ich es fuer unsere Hochzeit nutzen wuerde.

**Durchfuehrung**

- [ ] Testperson landet auf der Startseite ohne Vorerklaerung.
- [ ] Testperson erklaert mit eigenen Worten, was das Produkt ist.
- [ ] Testperson findet die Hauptaktion zum Erstellen eines Spaces.
- [ ] Testperson bemerkt den Demo-Einstieg.
- [ ] Testperson findet den Bereich `Mehr erfahren`.
- [ ] Testperson findet die FAQ ueber den Footer oder direkt auf der Seite.

**Erwartetes Ergebnis**

- [ ] Der Produktnutzen wird in unter 30 Sekunden korrekt verstanden.
- [ ] Die Begriffe `privater Link`, `Brautpaar-Bereich` und `QR-Code` werden intuitiv verstanden.
